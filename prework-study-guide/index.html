HTML

  • HTML stands for Hypertext Markup Language.
  • HTML provides the basic structure of the webpage. Picture an unpainted house with no plumbing, heating, etc.
  • The head element contains information about the webpage, including metadata.
  • The body element represents the visible content shown to the user.
  • Some elements are nested inside each other. The "parent" element holds "child" elements inside them.
  • HTML is case sensitive!!
  • HTML is the beginning element of a project, never the closing element.

CSS

  • CSS is used for content style.
  • There are three forms of CSS: Inline, Internal, and External. External is the most commonly used form and is accomplished my creating a separate CSS file and linking it to the 'head' of the HTML file
  • An importand CSS note for windows users when opening a new file, the touch command doesn't work. Instead use: type nul > file-name.css
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • A declaration contains two parts: the PROPERTY we want to apply, and the VALUE of the property
  • Ex: Declaration: Color(the property): Blue (the value);
  • For a hexidecimal color resource, copy this link: https://htmlcolorcodes.com/

Git

  • All git commands start with "git".
  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • A cheat sheet for making and closing a new branch:
  • cd name-of-directory-you-want-the-branch-under
  • git status main (this is to make sure you're on the main branch)
  • git pull origin main (this links github to your local workspace)
  • git checkout -b feature/new-branch-name
  • Make changes to your project. Be sure to save frequently!
  • Save your changes.
  • git add -A
  • git commit -m "breif description of the changes"
  • git pull origin main
  • git push origin feature/new-branch-name
  • Visit github to check for a push request.
  • git checkout main (brings you back to the main branch)
  • git pull origin main

JavaScript

  • JavaScript is what makes the website work.
  • Control flow is the order in which a computer executes code in a script.
  • The control flow for JavaScript is top to bottom.
  • A variable is a named container that allows us to store data in our code.
  • Primitive values are the lowest level of JavaScript.
  • These include: Null, Undefined, Boolean, Number, BigInt, String, and Symbol.
  • An array is a way to store a group of related data.
  • Loops perform a task on all items of an array by looping a block of code.
  • An If statement is used to interrupt the control flow in a loop.
  • Functions must be defined before you can call them.
  • To define a function, first declare the function using a keyword, second provide instructions in the code block, then provide a console log.
  • Once the function is defined, go to your console and type the keyword to call the function.
